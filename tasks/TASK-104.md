# TASK-104: Bulk Bundle Creation Detailed Design

## Objective
Create a detailed design for the bulk bundle creation APIs and supporting data model, covering BulkCreateBundles, GetSubmissionStatus, GetSubmissionHistory, and the SubmissionIndex DynamoDB table. This design should be detailed enough that implementation tasks can start immediately after review.

## Context
The PBXX→VBCS migration design (TASK-088) covers the high-level approach and deep-dives on GetBundleStatus + WorkflowState. The bulk APIs need their own detailed design because they introduce a new DynamoDB table (SubmissionIndex), a new submission grouping concept, and the 500-workflow fan-out handling.

**Table split:**
- **WorkflowState** → covered in TASK-088 design (backs GetBundleStatus, per-workflow status)
- **SubmissionIndex** → covered in THIS design (backs BulkCreateBundles, GetSubmissionStatus, GetSubmissionHistory, submission grouping)

**APIs in scope:**
- `BulkCreateBundles` — accepts structured payloads, writes SubmissionIndex, starts ORCA workflows
- `GetSubmissionStatus` — returns status of all bundles in a submission
- `GetSubmissionHistory` — returns all submissions for a user
- 500-workflow fan-out handling (parent ORCA workflow for >10 bundles)

**Not in scope (covered elsewhere):**
- GetBundleStatus / WorkflowState (TASK-088 design)
- DeleteBundle (TASK-073, MBWS deprecation design)
- ActivateBundle / SuppressBundleOffer (TASK-088 design — Limestone APIs)
- ABC client-side migration (TASK-096, blocked by this task)

## SubmissionIndex DynamoDB Table (from v3 appendix)

Tracks bulk submissions — maps a `submissionId` to its constituent ORCA workflow IDs.

| Attribute | Type | Key | Description |
|---|---|---|---|
| `submissionId` | String | PK | UUID generated by VBCS on `BulkCreateBundles` |
| `workflowIds` | List\<String\> | — | ORCA workflow IDs for each bundle in the submission |
| `submitterUsername` | String | GSI-PK | ABC/VCBM username who submitted |
| `marketplaceId` | String | GSI-SK | Marketplace (e.g., `ATVPDKIKX0DER`) |
| `clientId` | String | — | Calling service (ABC, VCBM, etc.) |
| `bundleCount` | Number | — | Total bundles in submission |
| `createDate` | String (ISO 8601) | — | Submission timestamp |
| `ttl` | Number | — | TTL epoch seconds (no TTL initially — PBMS has no TTL and clients see full history. Revisit after migration.) |

**GSI: `submitterUsername-index`**
- PK: `submitterUsername`
- SK: `createDate`
- Projection: ALL
- Purpose: `GetSubmissionHistory(username)` — returns all submissions for a user, sorted by date

**Access patterns:**

| Operation | Pattern | Cost |
|---|---|---|
| `BulkCreateBundles` | PutItem | 1 WCU |
| `GetSubmissionStatus(submissionId)` | GetItem → fan out to WorkflowState | 1 RCU + N status lookups |
| `GetSubmissionHistory(username, marketplaceId)` | Query GSI, filter on `marketplaceId` | ~1 RCU per page |

**Capacity:** On-demand. Expected volume: low (hundreds of submissions/day across all clients).
